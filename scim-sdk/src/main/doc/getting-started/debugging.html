Getting Started with the UnboundID SCIM SDK

Logging and Debugging

The UnboundID SCIM SDK provides a basic logging facility based on the JDK logging framework. Using this helps make it easier to maintain and debug applications written with the UnboundID SCIM SDK.

The primary class to use for logging and debugging is the com.unboundid.scim.sdk.Debug class. The SDK itself uses this class extensively to log exceptions and other various conditions. It exposes a number of static methods so that client code can make use of it as well. The following example demonstrates how to enable debugging and write information about all messages with a WARNING level or higher to a file named "/tmp/scim.log": 

Debug.setEnabled(true);
Logger logger = Debug.getLogger();
FileHandler fileHandler = new FileHandler("/tmp/scim.log");
fileHandler.setLevel(Level.WARNING);
logger.addHandler(fileHandler);

You can use that same Debug class to do any logging from within your application code. There is a generic debug() method which can be used in any context, and it has an overloaded version which accepts a Throwable as the last parameter. Using this in your exception handlers will make it much easier to track down problems.
