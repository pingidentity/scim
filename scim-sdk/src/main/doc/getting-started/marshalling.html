Marshalling and Unmarshalling Data

The com.unboundid.scim.sdk.marshal package contains types for converting SCIM objects to and from an external representation. Currently the UnboundID SCIM SDK supports marshalling via both XML and JSON representations. In addition, the BaseResource class implements a marshal(Marshaller, OutputStream) method with which you can serialize the resource using any marshaller you choose.

Custom marshaller implementations can easily be created by implementing the com.unboundid.scim.sdk.marshal.Marshaller and com.unboundid.scim.sdk.marhsal.Unmarshaller interfaces. These implementations are required to be thread-safe. Unmarshallers should typically make use of the com.unboundid.scim.sdk.SCIMAttribute and com.unboundid.scim.sdk.SCIMAttributeValue classes to convert the raw data back into Java objects. Likewise, marshallers will typically use the same classes to retrieve the raw data from the Java objects and convert in into the target format.
